1 xdrpp
2 =====
3 
4 This package consists of an
5 [RFC4506](http://tools.ietf.org/html/rfc4506) XDR compiler, `xdrc`,
6 and an [RFC5531](https://tools.ietf.org/html/rfc5531) RPC library
7 `libxdrpp` for C++11. It takes advantage of C++ to hide most of the
8 complexity of setting up connections and cleaning up data, and
9 provides a type-safe interface that should help avoid many programming
10 errors common with other C/C++ XDR/RPC libraries. The library
11 provides both synchronous (see [srpc.h](srpc_8h.html)) and
12 asynchronous/event-driven (see [arpc.h](arpc_8h.html)) interfaces to
13 RPC. A type-safe event harness allows integration with other file
14 descriptor, timer, signal, and inter-thread callbacks (see
15 [pollset.h](pollset_8h.html)). Deterministic marshaling to
16 byte-vectors makes it easy to hash or digitally sign XDR data
17 structures (see [marshal.h](marshal_8h.html)).
18 Other features include [pretty printing](printer_8h.html), tracing
19 (set the `XDR_TRACE_CLIENT` or `XDR_TRACE_SERVER` environment variable
20 or the corresponding lower-case global `bool` values), and optional
21 integration with [autocheck](autocheck_8h.html) and
22 [cereal](cereal_8h.html) (the latter of which can, among other things,
23 translate XDR to and from JSON).
24

32 A high-level example gives a flavor for the library. To use RPC you
33 first define a protocol by writing a `.x` file in
34 [XDR format](http://tools.ietf.org/html/rfc4506). Then, use the
35 included XDR compiler, [`xdrc`](md_doc_xdrc_81.html), to translate
36 this file into C++ types. Finally use XDRPP's facilities to translate
37 the XDR interface into a C++ interface on both ends: The client gets
38 an object with methods corresponding to the RPC procedures, while the
39 server must implement an object with methods for each of the
40 procedures.
41 
42 Here is an example XDR source file:
43 
44 ~~~~ {.c}
45 // myprog.x
46 
47 typedef string big_string<>;
48 
49 program MyProg {
50  version MyProg1 {
51  void null(void) = 0;
52  big_string hello(int) = 1;
53  big_string goodbye(big_string) = 2;
54  } = 1;
55 } = 0x2dee1645;
56 ~~~~
57 
58 Assuming a server that has registered with rpcbind, the client can
59 invoke these RPCs as follows:
60 
61 ~~~~ {.cxx}
62 // client.cc
63 // compile as: c++ -std=c++11 `pkg-config --cflags xdrpp` -c client.cc
64 // c++ -std=c++11 -o client client.o `pkg-config --libs xdrpp`
65 
66 #include <iostream>
67 #include <xdrpp/srpc.h>
68 #include "myprog.h"
69 
70 using namespace std;
71 using namespace xdr;
72 
73 int
74 main(int argc, char **argv)
75 {
76  unique_fd fd = tcp_connect_rpc("myserver.example.com",
77  MyProg1::program, MyProg1::version);
78  srpc_client<MyProg1> c{fd.get()};
79  unique_ptr<big_string> result = c.hello(5);
80  cout << "The result of hello(5) is " << *result << endl;
81  return 0;
82 }
83 ~~~~
84 
85 A server is not much more complicated, except that it must implement
86 each of the RPC methods as methods of a C++ class. For example:
87 
88 ~~~~ {.cxx}
89 #include <xdrpp/server.h>
90 #include "xdrpp/myprog.h"
91 
92 using namespace xdr;
93 
94 class MyProg1_server {
95 public:
96  using rpc_interface_type = MyProg1;
97 
98  void null();
99 
100  unique_ptr<big_string>
101  hello(unique_ptr<int> arg);
102 
103  unique_ptr<big_string>
104  goodbye(unique_ptr<big_string> arg);
105 };
106 
107 // ... implement three methods of MyProg1 ...
108 
109 int
110 main(int argc, char **argv)
111 {
112  MyProg1_server s;
113  srpc_tcp_listener rl;
114  rl.register_service(s);
115  rl.run(); // should never return
116  return 1;
117 }
118 ~~~~
119 
120 The compiler has options (`-serverhh` and `-servercc`) to generate
121 scaffolding for `class MyProg` and its declaration.
122 
123 Note that these examples assume the server is running `rpcbind`, so
124 that the client can find the TCP port number of the RPC program on the
125 server. It is also possible to connect manually and pass file
126 descriptors for a connected socket to the constructor for
127 `srpc_client`, or to pass a `unique_fd` to an `srpc_tcp_listener` (in
128 which case the `srpc_tcp_listener` takes ownership of the file
129 descriptor).
